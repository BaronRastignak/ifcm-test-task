# IncomingFileDetector

## Задача:

Разработать тестовое приложение на C# в ОС Windows, которое фиксирует момент поступления в директорию любых файлов и выводит информацию об имени файла и времени его прихода.

Для разработки выбран фреймворк WPF.

## Требования

- Отслеживание поступающих файлов должно выполняться в отдельном потоке.
- Функциональность отслеживания должна быть отделяемой от WPF приложения для использования в других приложениях.
- Путь к директории входных файлов должен быть настраиваемым.
- Приложение должно логировать все операции.
- Код должен использовать современные возможности .NET и быть комментированным.

## Особенности реализации

- Функциональность отслеживания файлов покрыта тестами.
- Настройки пути к директории входных файлов и таймаута отслеживания вынесены в пользовательский интерфейс.
- При разработке использован подход Dependency Injection для отделения функциональности отслеживания файлов от логики GUI.

## Что можно доработать

- Уточнить правила отслеживания дубликатов файлов (сейчас - по имени файла; можно определять дубликаты по хэш-сумме содержимого или по хэш-сумме И имени файла).
- Сделать приложение мультиязычным, с английской и русской локалями.
- Реализовать сохранение пользовательских настроек папки и таймаута отслеживания при закрытии приложения.
- Улучшить тестовое покрытие для граничных случаев.
- Улучшить обработку ошибок и исключений (не только выводить их в лог, но и оповещать пользователя; к примеру, просить сменить директорию для отслеживания, если к выбранной отсутствует доступ).
